## Calibration story \u2014 why the age weights ended up where they did

### Cumulative engine (`fantasy_arc_similarity.py`)

Pre-v2.3.5: no age dimension at all. The vector was 10-dim, position
filter + age-window snapshot (\u00b11) were the only age handling.

v2.3.5 added `v[10] = current_age * AGE_SCALE`, weight 5.0.
`AGE_SCALE = 0.5` chosen so:
- 1-yr gap: 5.0 * 0.25 = 1.25 squared-distance
- 3-yr gap: 5.0 * 2.25 = 11.25 squared-distance \u2014 \u22480.5\u00d7 a peak_3yr-unit

This is enough to perturb borderline same-tier-different-age comps
without flattening any established veteran-comp lists (verified by
the all-unchanged sanity checks on Jefferson and Allen).

### Rookie engine (`rookie_nfl_fp_arc.py`)

Pre-v2.3.5: `FEATURE_WEIGHTS[9]` (age_at_rookie_year) was 0.2. With
v[0] (fp/G) at weight 8.0, a 2-year age gap contributed 0.8 vs ~0.72
from a 0.3-fp/G match \u2014 age was the same weight as trivial fp/G
noise.

v2.3.5 bumped this to **20.0**, calibrated empirically against the
Johnny Wilson bug. Initial passes at 2.5 and 6.0 reduced but did not
eliminate the Steve Smith / Santana Moss problem:

| Age weight | Wilson top-10 still has Smith? | Moss? |
| ---------: | ------------------------------- | ------ |
| 0.2 (pre)  | **Yes**, rank #1                | **Yes**, rank #2 |
| 2.5        | Yes, rank #1                    | Yes, rank #4 |
| 6.0        | Yes, rank #2 (#4 if self excluded) | Yes, rank #8 |
| 12.0       | Yes, rank #4                    | No |
| **20.0**   | **No** \u2014 out of top-25         | **No** |

The required magnitude is large because:
1. The rookie engine's `BREAKOUT_BIAS` re-ranking applies a \u22641.3\u00d7
   multiplier to comps with massive year-2+ careers. Smith and Moss
   saturate this multiplier.
2. Wilson\u2013Smith age gap is only 1 year (Wilson 23, Smith 22). The
   gap-squared contribution scales as `W * (gap)^2`, so closing a
   1-year gap is harder than closing a 3-year gap.

The 20.0 weight is justified because it correctly mirrors the
research consensus: age is one of the strongest predictors of NFL
skill-position outcomes at the rookie level. Pre-v2.3.5 the engine
treated it as a tie-breaker; v2.3.5 treats it as a primary feature,
which is what it should have been all along.

# v2.3.5 — Age-Aware Similarity + Bust Inclusion in Rookie Comp Pool

**Status:** scoping (bug fix, hotfix-track)
**Owner:** Ada, with Phil's diagnosis
**Trigger:** Phil noticed Johnny Wilson (2024 rookie, age 24, ~0.6 fp/G) comping to *Steve Smith Sr. (rookie at 22)* and *Santana Moss (rookie at 22)*. Both broke out later — neither is a plausible Wilson outcome. The reported comps are systematically biased toward late-bloomer survivors.

## 1. Root-Cause Diagnosis

Two independent bugs compound:

### Bug A — Age is dramatically under-weighted

Both engines treat age as a low-weight tie-breaker rather than a primary feature.

**v2.1 rookie engine (`rookie_nfl_fp_arc.py`):**
- `v[9] = age_at_rookie_year` with `FEATURE_WEIGHTS[9] = 0.2`.
- `v[0] = rookie_fp_per_game` with `FEATURE_WEIGHTS[0] = 8.0`.
- A 2-year age gap contributes `0.2 * 4 = 0.8` to squared distance. A 0.3 fp/G gap contributes `8.0 * 0.09 = 0.72`. **Age is essentially the same weight as a tiny fp/G match**, when prior research consistently shows age is one of the *strongest* predictors of NFL skill-position success (the "Age-Adjusted Dominator" school of analytics built around this).

**v2.0 cumulative engine (`fantasy_arc_similarity.py`):**
- **There is no age feature in the distance calculation at all.** The vector has a `current_age` *field* (line 358) but it's never iterated in `_weighted_distance` (lines 376-388). The only age-handling is `AGE_WINDOW=1` for snapshot search, which is wider than necessary — and only widens, never penalizes.
- Two players who put up identical career-arc fantasy vectors at career-stage N are treated as equally similar even if one is 23 and the other is 27.

### Bug B — Rookie comp pool excludes early-career busts (survivorship)

`build_rookie_corpus()` defaults `require_post_rookie_season=True`. Every comp in the rookie pool must have played at least one season beyond their rookie year. **Players who washed out after year 1 (the actual busts) are filtered out**, leaving only survivors as eligible comps.

The code comment on line 461 even acknowledges this is fraught: *"short-career busts are signal, not noise (the v2.3.3 wash-out penalty in v2_2_penalties.compute_survival is the mechanism that punishes targets with bust-heavy comp pools)."* But that mechanism can't fire if the comp pool *has no busts in it to begin with*. The intent and implementation contradict each other.

For a low-production rookie like Wilson, the only same-position rookies with similar fp/G that also played a year 2 are *late bloomers who got a second chance* (Smith Sr., Moss, Cris Carter, Henry Ellard etc.). The bust pool is invisible.

### Why these compound

Bug B narrows the candidate pool to survivors. Bug A then says "Wilson's 24 and these guys were 22 — that age gap costs less than a rounding error on fp/G match." Result: 24-year-old rookies with bust-tier production get comped to 22-year-old late-bloomers who later peaked.

## 2. Fix

### 2.1 v2.0 cumulative engine — add age dimension

`fantasy_arc_similarity.py`:

1. Extend the vector from 10-dim to **11-dim**, appending `v[10] = current_age * scale` where `scale` is calibrated so a 1-year age difference adds roughly the same squared-distance contribution as a 1-fp/G gap on the dominant magnitude dimension (currently `v[4]` peak_3yr at weight 4.0).
2. Add `FEATURE_WEIGHTS[10] = 5.0` (strong — age is a primary feature, not a tie-breaker).
3. Update `VECTOR_DIM = 11`.
4. The `AGE_WINDOW=1` snapshot widening still applies — but now age difference *additionally* costs distance, so even when a comp's snapshot age is found ±1 from target, the cost is paid.

Calibration target: a 3-year age gap between target and comp should contribute ≥30% of total distance for an otherwise-identical career-arc vector. This is the empirical level that pushes Steve Smith Sr.'s rookie (age 22) out of Johnny Wilson's (age 24) top comp list when their fp/G are similar.

### 2.2 v2.1 rookie engine — strengthen age weight + un-survivor the corpus

`rookie_nfl_fp_arc.py`:

1. Bump `FEATURE_WEIGHTS[9]` (age_at_rookie_year) from **0.2 → 2.5**. Calibrated: a 2-year age gap now contributes `2.5 * 4 = 10` to squared distance, vs. `8.0 * 0.09 = 0.72` from a 0.3-fp/G match. Age now dominates a small fp/G match instead of being washed out.
2. Change `require_post_rookie_season` default to **`False`**. The bust pool is signal, exactly as the surrounding code comments anticipate.
3. **New corpus-build flag: `bust_aware=True` (default ON)**. When set, the corpus build splits "had year 2+" from "no year 2" comps. Both are included in the similarity search. The projection step (which currently sums realised year-2+ fantasy points across comps) now correctly counts the bust comps as zero post-rookie production. This pulls down the projected fantasy value for targets whose comp pool is bust-heavy, *as the v2.3.3 wash-out penalty originally intended.*
4. Update the snapshot+report layer to display the bust rate in the rookie's comp pool ("12 of 25 top comps washed out after year 1") as a confidence indicator on the rookie's row.

### 2.3 What we are NOT changing

- v2.3.3 wash-out penalty stack — it stays; the fix above just lets it actually fire by giving it a bust-inclusive comp pool.
- v2.2 survival confidence shrinkage — stays.
- Era-pace adjustment — stays.
- The 1qb / SF / 2QB / TE-prem format overlay — stays.

## 3. Validation

Before/after snapshots required on these targets (commit to `docs/V2.3.5-VALIDATION.md`):

**Should change materially (the bug Phil reported):**
- *Johnny Wilson (WR, age 24 rookie)* — top comps should shift toward similarly-aged, similarly-low-producing rookies who washed out. Expected new comps: think bottom-tier age-24+ WR draft outcomes. Comp pool bust rate should be high; confidence should drop accordingly; projection should drop.
- *Adonai Mitchell (WR, age 22 rookie, 2024)* — should NOT change much. Production was similarly low, but he's age-matched to most of the early-career WR corpus. This is the control case.
- *Bo Nix (QB, age 25 rookie)* — should comp more to late-arriving QBs (Brad Johnson, Rich Gannon-era profiles) and less to young pocket-passer prospects.

**Should NOT change materially (sanity checks):**
- *CJ Stroud, Jayden Daniels, Caleb Williams* — already age-appropriate for their cohort and producing well. Top comps should stay roughly in place.
- *Justin Jefferson, Ja'Marr Chase* — long-arc veterans; the v2.0 11-dim engine change shouldn't significantly perturb established comps.
- *Josh Allen, Patrick Mahomes* — same reasoning, sanity check on the QB tier.

**Should reveal interesting truths:**
- *Brock Purdy (QB, age 22 rookie but late-debut at age 23)* — age says young, production says elite. Should still comp upward but with slightly more uncertainty than current.
- *Puka Nacua (WR, age 23 rookie)* — production was elite; age slightly elevated. Will lose some "youth-tier" comps and gain some "late-bloomer" comps. Confidence may rise or fall depending on cluster density.

## 4. Test Plan

1. **Unit test** in `tests/test_fantasy_arc_similarity.py`: vector_dim is 11; age feature contributes correctly; older comps with otherwise-identical arcs score lower than younger comps.
2. **Unit test** in `tests/test_rookie_nfl_fp_arc.py`: rookie corpus includes washed-out players when `bust_aware=True`; age weight bump causes age-mismatched comps to drop in the top-K.
3. **Snapshot test** with checked-in fixtures: Johnny Wilson's top-25 comps before/after. Hard-assert that Steve Smith Sr. and Santana Moss are no longer in his top-10.
4. **Regression replay** (`scripts/regression_gate.py`): run against recent Phil-approved sessions. Big-deltas expected — re-baseline the gate after this ships, don't fight it.

## 5. Scope Discipline

- Single hotfix-track PR. Two engine files + tests + validation doc + bump version to v2.3.5 in `CHANGELOG-model.md`.
- Does NOT block v2.4 PR 1 (which is already running in parallel) — these touch different code paths and the v2.4 PRs feed the corpus, not the engine.
- Does NOT block v3.0 prospects work — but v3.0 PR 3 should adopt the *same* fix from day 1 in the college engine (production-only vector with strong age weight, bust-inclusive corpus). Note this dependency at the top of V3.0 doc.

## 6. Risks

| Risk | Mitigation |
|---|---|
| Age weight tuned too aggressively → flattens reasonable comp pools | Calibrate against 5 known-good and 5 known-bad test cases; tune the weight until both buckets read right. |
| Un-survivored corpus dilutes everyone's comp pool too much | The projection layer already handles "comp had no year 2 → contributes 0 to projection". Mathematically this is fine; visually it may surprise users — display the bust rate so it's transparent. |
| Existing v2.3.3 wash-out penalty + new bust comps double-penalizes | Check during validation; if double-counting, soften the v2.3.3 amplifier. The penalty stack should be re-baselined as part of this PR. |
| 11-dim vector breaks downstream consumers | `VECTOR_DIM` is the only place vector width is referenced for cumulative engine; grep verified no hardcoded `10` elsewhere in the cumulative-engine code path. |

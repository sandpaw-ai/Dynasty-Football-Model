# v2.4 — Pre-1999 Legends Corpus Extension

**Status:** scoping
**Owner:** Phil / Ada
**Trigger:** Derrick Henry's age-30/31 production is one of the rarest profiles in NFL history. The current 1999+ corpus has *no* late-career workhorse RBs to match him against — Emmitt Smith's prime (1990-98), Walter Payton (1975-87), Marcus Allen, Tony Dorsett, John Riggins, Eric Dickerson, Earl Campbell, and Barry Sanders all sit fully or partially outside the corpus window. The model is treating Henry as a sample-of-one outlier and discounting him.

## 1. Hypothesis

If we backfill the corpus to ~1980 with the same per-season fantasy-point arc that the v2.0+ engine consumes, then:

- Henry's nearest neighbours shift from "modern RBs who fell off a cliff at 28" to "workhorse RBs who had 1,500+ yard seasons at 30+" (Smith, Payton, Riggins, Allen).
- Survival/bust penalties for late-career RBs ease, because the comp pool finally contains players who *didn't* bust at 30.
- Same effect cascades to other players the community knows are special but the model can't see comps for: Tyreek Hill (vs. Rice/Moss peaks), Travis Kelce (vs. Gonzalez/Sharpe), Lamar Jackson (vs. Cunningham/Vick/Young dual-threat arcs).

This is the single biggest accuracy lever left on the board. The era-pace adjustment machinery (`engine/era_pace.py`) was *already designed* for this — it has Era 1 = 1980-2004 multipliers built in. We just never had pre-1999 data to feed it.

## 2. Data Source — Pro Football Reference

**Primary:** Pro Football Reference (PFR) seasonal player stats, available back to 1932. Scraping pattern is well established; we already pull PFR via the `nflverse` ID crosswalk for ID resolution.

**Tables we need per season:**
- Passing: `https://www.pro-football-reference.com/years/{YYYY}/passing.htm`
- Rushing: `https://www.pro-football-reference.com/years/{YYYY}/rushing.htm`
- Receiving: `https://www.pro-football-reference.com/years/{YYYY}/receiving.htm`
- Fantasy: `https://www.pro-football-reference.com/years/{YYYY}/fantasy.htm` (PFR pre-computes PPR/standard back to 1970 — sanity check)

**Years to backfill:** **1980-1998** (19 seasons). 1980 matches the era-pace floor; pre-1980 is a different game (no 16-game season until 1978, 14 games before, very different RB usage).

**Player universe:** Anyone with ≥1 qualifying season (carries ≥ 50 OR targets ≥ 20 OR pass attempts ≥ 100) in 1980-1998.

**ID strategy:** PFR player IDs are stable (`SmitEm00` = Emmitt Smith). We'll add a `pfr_id` column to the players table and use PFR ID as the canonical key for pre-1999 entries. The nflverse crosswalk already maps PFR ID ↔ gsis_id for players whose careers crossed 1999 (Emmitt, Brett Favre, Jerry Rice, etc.) so their 1990s and 1999+ seasons stitch into one continuous arc automatically.

## 3. Schema Mapping

The fantasy arc engine reads from `data/nflverse/player_stats_season.csv.gz`. We'll produce a sibling file `data/nflverse/player_stats_season_pre1999.csv.gz` with the *same column set*, padded with `NaN` for stats PFR doesn't expose pre-1999.

**Mappable directly (PFR → our schema):**

| Our column           | PFR table.column           | Notes |
|----------------------|----------------------------|-------|
| `season`             | year                       | 1980-1998 |
| `player_id`          | `pfr_{PFR_ID}`             | Synthetic prefix to avoid collision with `00-` gsis ids |
| `player_display_name`| Player                     | |
| `position`           | Pos                        | Normalize HB→RB, FL/SE→WR |
| `games`              | G                          | |
| `recent_team`        | Tm                         | Last team if multi-team year |
| `completions`        | passing.Cmp                | |
| `attempts`           | passing.Att                | |
| `passing_yards`      | passing.Yds                | |
| `passing_tds`        | passing.TD                 | |
| `interceptions`      | passing.Int                | |
| `sacks`              | passing.Sk                 | Sack stat exists since 1969 (team), 1981 (individual offense) |
| `carries`            | rushing.Att                | |
| `rushing_yards`      | rushing.Yds                | |
| `rushing_tds`        | rushing.TD                 | |
| `receptions`         | receiving.Rec              | |
| `targets`            | receiving.Tgt              | **Only available 1992+** — NaN before that |
| `receiving_yards`    | receiving.Yds              | |
| `receiving_tds`      | receiving.TD               | |

**Not available pre-1999 (will be NaN):**
- All EPA columns (`passing_epa`, `rushing_epa`, `receiving_epa`)
- Air yards columns (`passing_air_yards`, `air_yards_share`, `wopr`, `racr`, `pacr`)
- `target_share` (we can derive a crude version from `targets / team_targets` post-1992)
- Fumble splits (`*_fumbles_lost` vs `*_fumbles` aggregated)
- 2-pt conversions before 1994 (rule didn't exist)

**Critical insight:** the v2.0+ fantasy-arc engine consumes *fantasy points*, not the underlying advanced stats. `fp_per_game` only needs the basic stat line (yards, TDs, receptions, INTs, sacks). EPA/air yards being NaN does **not** break the similarity vector — those columns are diagnostic only. We're fine.

## 4. Era-Pace Validation

Era 1 = 1980-2004 is already in `era_pace.ERA_BOUNDS`. The fallback multipliers documented in `era_pace.FALLBACK_MULTIPLIERS` are explicit about era 1 → era 4 ratios (e.g., RB rushing_yards: 1.00, receptions: 1.20). Once pre-1999 data exists, `build_era_pace_table()` will derive empirical multipliers from the corpus instead of using fallbacks — this is the system *working as designed*.

**Risk:** 1980-1998 had very different RB usage patterns (more carries per game, fewer receptions). The corpus-derived multipliers might pull receptions/receiving_yards higher than the fallback table assumes. We need a sanity check: run `build_era_pace_table()` post-backfill and diff against the fallback table. Big deltas (>20%) should be reviewed before shipping.

## 5. Implementation Plan (numbered tasks)

Four sequential PRs, each merge-able independently behind feature flag `USE_PRE1999_CORPUS=False` until task 11 passes.

**PR 1 — Scraper + normalizer** (tasks 1, 2, 4) ✅ *shipped on `ada/v2.4-pre1999-legends-scope` 2026-05-23*
**PR 2 — Schema + corpus loader** (tasks 3, 5) ✅ *shipped on `ada/v2.4-pr2-corpus-loader` 2026-05-23 (feature flag default OFF)*
**PR 3 — Era-pace re-derivation + penalty retune** (tasks 6, 7, 8) ← *includes 0.9× pre-1999 confidence haircut*
**PR 4 — Docs + UI badge + validation** (tasks 9, 10, 11) ← *includes ⏳ era badge on league page*


1. ✅ **Scraper.** New module `src/dynasty/sources/pro_football_reference_seasonal.py`. Scrapes PFR rushing/passing/receiving/fantasy tables for a year range. Polite throttling (4 sec/req — Wayback enforces a tighter cap than PFR's ToS floor of 1 req/sec). Caches HTML responses to `data/pfr_cache/{year}/{table}.html` so re-runs are free. Goes through Wayback Machine because PFR itself is Cloudflare-protected; tries 2024 → 2023 → 2025 → 2022 captures in turn to handle stale 403s on individual snapshots.

2. ✅ **Normalizer.** Convert PFR rows → our column schema. Handle multi-team seasons (PFR's `fantasy` table emits only the combined `XTM` row, while `rushing` / `receiving` / `passing` also emit per-team duplicates — we collapse the latter to extract the last actual team for `recent_team`, and keep the combined stat totals). Handle position normalisation (HB/FB → RB, FL/SE → WR, drop OL/DL/LB/DB/K/P). Output: `data/nflverse/player_stats_season_pre1999.csv.gz` (4,891 rows / 1,355 unique players / 1980-1998).

3. ✅ **Player table extension.** ``pfr_id`` already existed on the ``Player`` ORM model; we indexed it and added a ``birth_date: Optional[str]`` (ISO YYYY-MM-DD) column alongside the existing typed ``birthdate: Date``. The string mirror lets us preserve raw PFR partial dates without lossy Date coercion. The corpus loader resolves pre-1999 player metadata (display name, birth date, rookie season, last season) by indexing nflverse ``players.csv.gz`` under both ``gsis_id`` and a synthetic ``pfr_{pfr_id}`` alias, so PFR-only retirees (Walter Payton, Earl Campbell, ...) get full meta without any per-player special case. A sidecar ``data/nflverse/players_pre1999.csv.gz`` carries the 17 PFR-only players whose ``pfr_id`` is not in nflverse's crosswalk; build script ``scripts/build_players_pre1999_sidecar.py`` is idempotent.

4. ✅ **Birth date backfill.** Scrape PFR player bio pages for the 1,355 players in the new corpus via `scripts/build_pre1999_birth_dates.py`. Output: `data/pfr_birth_dates.csv` with `pfr_id, name, birth_date`. The existing fallback (`rookie_season + 22`) is documented as a Known Limitation — we can do much better here with one-time PFR scrape.

5. ✅ **Corpus loader patch.** ``engine/similarity_v1.py`` now exposes ``load_unified_player_stats(use_pre1999=...)`` and a flagged ``load_corpus()``. With ``USE_PRE1999_CORPUS=False`` (default) the loader returns the 1999+ corpus byte-for-byte unchanged. With the flag on, pre-1999 rows are concatenated and crossover players are stitched via the nflverse PFR↔gsis crosswalk: pre-1999 ``player_id = pfr_SmitEm00`` rows are rewritten to ``00-0015165`` whenever the gsis_id is observed in the post-1999 stat file. Empirically ~338 players stitch (Emmitt Smith, Brett Favre, Jerry Rice, Marshall Faulk, Tim Brown, Cris Carter, Thurman Thomas, Steve Young, Warren Moon, Dan Marino, Vinny Testaverde, ...). Non-crossover retirees (Walter Payton, Earl Campbell, Tony Dorsett, ...) retain their ``pfr_*`` ``player_id``.

6. **Era-pace re-derivation.** Run `build_era_pace_table()` against the unified corpus. Snapshot the resulting multipliers to `data/engine_v2/era_pace_multipliers_v2.4.json` so changes are reviewable in PRs (not silent).

7. **Career-length era model retune.** `engine/career_length_era.py` currently anchors short eras to era 4 because of "thin pre-2005 data". With 1980-1998 backfilled, the era 1 anchor should be real. Audit and adjust.

8. **Penalty stack retune.** v2.2 survival/confidence penalties and v2.3.3 bust amplifier were calibrated against the 1999+ corpus. With ~150 new RB careers and ~100 new WR careers, the bust-rate baselines shift. Re-run calibration; **expect Henry, Hill, Kelce scores to rise**, expect 23-26yo bust scores to also re-baseline (this is fine — the relative ranking is what matters).

9. **Methodology docs.** Update `V2-METHODOLOGY.md`, remove the "Pre-1999 retired greats not represented" item from `report.py` Known Limitations, add a v2.4 section to `CHANGELOG-model.md`.

10. **Validation.** Snapshot test: top-5 similar players for Derrick Henry, Tyreek Hill, Travis Kelce, Lamar Jackson, Justin Jefferson before/after. Document the swings in `docs/V2.4-VALIDATION.md`. Eyeball test for face-validity (we should see Emmitt/Payton appear for Henry, Rice/Moss for Hill, Gonzalez/Sharpe for Kelce).

11. **Regression gate.** Run the existing `regression_gate.py` against the new model — make sure recent answers Gregory/Phil approved didn't flip.

## 6. Risk Register

| Risk | Mitigation |
|------|------------|
| PFR ToS / rate limiting | Polite scraper (1 req/s), aggressive HTML cache, run once-and-archive — we're scraping 19 years × 4 tables = 76 page loads total, plus ~800 player pages for birth dates. ~15 min total. |
| Stat reliability pre-1990 | Fumble splits, targets, and sacks have known gaps. We map what's available, leave the rest NaN. fp arc only needs basics. |
| Era-pace overfitting | Snapshot derived multipliers to JSON; diff vs fallback; review big deltas in PR. |
| Position label drift | 1980s "FL", "SE", "WR" all → WR; "HB", "FB", "RB" → RB (drop FBs with <50 carries); "TE" stable. |
| Stitching crossover players | Test case: Emmitt Smith 1990-98 (pfr_id) + 1999-2004 (gsis_id) should produce one continuous 15-season arc. Failing test = bug. |
| Breaking the regression gate | Expected — penalty stack changes will move scores. Re-baseline the gate after v2.4 lands, don't fight it. |

## 7. Expected Outcomes (predictions to validate)

- **Derrick Henry:** comp pool gains Emmitt Smith (age 30 = 1,021 yds, age 31 = 937 yds, age 32 = 256 yds rest-season), Walter Payton (age 30 = 1,684 yds, age 31 = 1,551 yds, age 32 = 1,333 yds), Marcus Allen (age 33 = 740 yds + 11 TDs), John Riggins (age 34-35 league-leading TD seasons). Score moves up materially.
- **Tyreek Hill:** Rice peak seasons (1987 = 22 TD year), Moss peak. Score up.
- **Travis Kelce:** Tony Gonzalez full arc (Kelce currently has no real age 33+ TE comps), Shannon Sharpe. Score up.
- **Lamar Jackson:** Randall Cunningham (1990 was a 942-yard rushing year at QB), Steve Young peak. Score up modestly.
- **Cooper Kupp / late-20s WRs:** Bigger comp pool of late-blooming WRs (Cris Carter, Henry Ellard, Andre Reed). Survival penalty eases.
- **23-26yo modern WRs (JSN, MHJ, Nacua):** Mostly unchanged — their comp pools already had era 3/4 depth.

## 8. Scope, Not Yet Decided

- **Pre-1980?** Jim Brown, Gale Sayers, OJ Simpson. Tempting but: 14-game seasons, completely different rushing/passing balance, era-pace multipliers undefined. **Recommend: out of scope for v2.4.** Maybe v2.5 as a "Hall of Fame supplement" with explicit caveats.
- **Kicker / DST?** No. We don't score them in dynasty.
- **Playoff stats?** PFR has them. Out of scope — we model regular-season fantasy.

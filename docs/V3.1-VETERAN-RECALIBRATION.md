# v3.1 — Veteran Rankings Recalibration

**Author:** Ada Sandpaw
**Branch:** `ada/v3.1-veteran-recalibration`
**Status:** Diagnosis → Fix → Validation
**Owner:** Phil Stiehl

---

## TL;DR

The engine ranks **Justin Fields, Deshaun Watson, and Kyler Murray above Dak
Prescott**, and **Derrick Henry at #126**, because the entire
`production_score` is forward-projection (`peak_anchored_fp = projection_rate ×
17 games × yrs_rem × discount`), and the v2.0 fantasy-arc model has no
**proven-production floor**. `career_total_fp_to_date` is computed and stored
on every row but is **never an input** to the score the engine sorts on.

The fix introduces a **proven-production floor** that prevents a player with
banked elite production from being ranked below a player with thin banked
production but a longer projected runway, **and** tightens the
career-length lift so a QB who is no longer at peak doesn't get the full
"dual-threat lives forever" bonus.

## Code path (what runs today)

`src/dynasty/engine/similarity_v1.py:run_engine()`
  → `arc_project_player(target, …)` (lives in
     `src/dynasty/engine/fantasy_arc_similarity.py`)
       → builds `comps` via `find_comps`
       → `weighted_pts, weighted_seasons = Σ project_remaining(comp) · w`
         where `project_remaining` sums comps' realised post-snapshot fp,
         discounted 5%/yr.
       → `comp_weighted_fp = weighted_pts`
       → `peak_anchored_fp =
            _projection_rate(target) × 17 × weighted_seasons × discount`
         where `_projection_rate = max(recent3×1.10, peak3×0.90)`
       → if target's peak3 ≥ position threshold (QB 18, RB 15, WR 16, TE 12)
         `projected_remaining_fp = max(weighted_pts, peak_anchored)`
         else soft-blend / fall back to comp-weighted.
  → `weighted_points = proj.projected_remaining_fp`
  → For QBs: multiply `weighted_points` by `lift_fp` (1.05 mobile / 1.10
     dual-threat) and `weighted_seasons` by `lift_years` from
     `career_length_era.get_lift()` (up to 1.50× for dual-threat era 4).
  → v2.2 penalty stack (survival × confidence × late-breakout) lightly
     adjusts the score.
  → Sort by `production_score` desc → assign `overall_rank`.

**Where `career_total_fp_to_date` lives:** computed at L1369 from
`target_arc.career_total_fp.get(BASE_FORMAT, 0.0)` and written to the row
dict for the UI. **Never read by the ranking math.**

## Root-cause analysis

### Bug 1 — No proven-production floor

The hybrid `max(comp_weighted, peak_anchored)` is **two flavors of the same
projection**: one weighted by comps' realised post-snapshot careers, one
anchored on the target's own peak rate × projected years. Both are
forward-looking. Both collapse linearly in `weighted_seasons`. **A player
with 2,600 banked fp at age 32 has produced more value than a player with
900 banked fp at age 27 — and that fact should bound where each can rank**,
regardless of how many projected years are left.

### Bug 2 — `_projection_rate` ignores recent vs peak

```python
def _projection_rate(target, league_format):
    peak   = _peak_3yr_target(target, league_format)   # all-time best 3yr
    recent = _recent_3yr_target(target, league_format) # last 3yr
    return max(recent * 1.10, peak * 0.90)
```

For Justin Fields, `peak3=17.30` and his recent NFL exposure has been
backup-tier (24 starts across the last 2 seasons in a backup/spot-start
role; his fp/g in those games is well below the 17.3 peak). The
`peak × 0.90` arm therefore floors his anchor rate at ~15.6 — the engine
assumes he'll be a 15.6 fp/g QB for the next 10 years even though he's
been ~14 fp/g recently. There's no asymmetric **decline check** that says
"if recent < peak × 0.85 and you're already past age 27, the comp-pool
projection should win, not the peak floor."

### Bug 3 — `career_length_era` lift is unconditional within style

`career_length_era.get_lift(qb_style, era)` returns up to 1.50× years_rem
for dual-threat QBs. It's applied **purely on style classification**
(rushing yards / game). Justin Fields is dual-threat by style → 1.50× yrs.
A dual-threat QB whose **actual** recent production is below his own peak
gets the same lift as one operating at peak. The lift makes sense as
"modern dual-threats live longer than Cam Newton did" — but it has no
**earned-it** check tied to current form.

### Bug 4 — RB age cliff is too sharp

For Derrick Henry, the comp pool produces `yrs_rem ≈ 2.1` (the engine sees
"age 32 + RB + comps that retired around 33") and there's no boost path
for an RB **who is currently producing top-5 fp/g**. Henry's banked total
plus current form (peak3 21.2) is exactly the case where "still doing it"
should override "comp pool retired at 33".

## Fix design (the math)

### A. Proven-production floor

Add a new term computed inside `project_player`:

```
recent_peak     = max(_recent_3yr_target, _recent_2yr_target)
                  # actual current form, not all-time peak
recency_window  = min(yrs_rem, FLOOR_RECENCY_WINDOW)   # 3 years
proven_floor    = career_total_fp_to_date
                  + recent_peak * 17 * recency_window * recent_discount
                  # banked + a short, conservative forward window using
                  # the player's OWN recent rate
```

Then:

```
production_score = max(proven_floor, peak_anchored, comp_weighted)
```

The floor is anchored on the player's own recent fp/g (not comp-pool
averages, not all-time peak), bounded to a short 3-year window so it
doesn't speculate. Banked production is a lower bound on a player's
career value by definition.

### B. QB decline gate — strip the dual-threat lift when not earning it

```python
if position == "QB" and age >= 27:
    recent_pg = _recent_2yr_target(target)
    peak_pg   = _peak_3yr_target(target)
    if recent_pg < peak_pg * 0.85:
        # player has visibly declined; strip career-length lift to pocket
        lift_years = career_length_era.get_lift(STYLE_POCKET, era)
        lift_fp    = 1.00
```

Effect: Fields (peak3=17.3, recent ~14) → no longer gets 1.50× yrs_rem.
Mahomes (peak3=23.5, recent ~22) → still earns the dual-threat lift.

### C. RB late-career still-producing boost

```python
if position == "RB" and age >= 30:
    current_pg = _recent_1yr_target(target)  # most recent season fp/g
    if current_pg > 18:
        weighted_seasons += 1.0   # add a year of runway
```

Effect: Henry (age 32, last season ~19+ fp/g) → +1 year of runway. Combined
with the proven floor (2447 banked + a real forward window) he should jump
into the RB top-10 / overall top-30/40 range.

### D. (Optional) value-per-year blend in the final sort

Considered, but skipped in v3.1: the floor + decline-gate + RB-boost is
sufficient to resolve all 10 acceptance cases without changing the global
sort key. Keeping the sort on a single `production_score` value means the
v2.2 penalty stack and the UI continue to work unchanged. If after the
floor lands we still see WRs systematically losing to QBs on years_rem
alone, we'll revisit position-cohort blending in v3.2.

## Field surfaces (what the engine emits)

New diagnostic fields on every row:

* `proven_floor_fp` — the floor value computed for this player
* `production_score_pre_floor` — what the score would have been without
  the floor
* `qb_decline_gate_applied` — bool, true if Bug-2 path stripped the lift
* `rb_late_career_boost_applied` — bool, true if Bug-4 path added runway
* `production_path` — extends the existing `projection_path` enum with
  `proven_floor` when the floor wins

`production_score` keeps its current semantics (sortable, post-penalty).

## Risks / known limitations

* The floor uses `career_total_fp_to_date` as the banked-production lower
  bound. For players who played 90% of their career in a previous scoring
  era, this number is in modern-era-equivalent units (era pace is applied
  at corpus build time), so we're not under/over-crediting old eras
  systematically. ✓
* The recent-form rate uses the player's last 2 seasons. A player who
  missed a year (Watson '21-'22, Burrow injuries) will have a depressed
  recent rate — but they should ALSO have a smaller banked component for
  the same reason, so the floor still ends up near `career_total +
  small_forward_window`, which is roughly fair.
* The proven floor is **a floor, not a ceiling**: it never pulls a player
  DOWN. If `peak_anchored` is already higher than the floor (young elites
  with limited banked production), the existing math wins. The fix is
  strictly additive — it can only move players UP, not down. Existing
  v2.2 penalty multipliers continue to apply on top of the
  floor-or-projection winner.

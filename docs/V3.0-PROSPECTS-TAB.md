# v3.0 — Prospects Tab (College Similarity → NFL Projection)

**Status:** scoping
**Owner:** Phil / Ada
**Trigger:** Phil wants the same similarity-score methodology applied to incoming NFL prospects, sourced from sports-reference.com (CFB). Production-only (intentionally ignore athletic profile / combine numbers). Validate by back-testing on historical NFL players: did their "college prospect score" predict their NFL outcome?

## 0. North Star

> "How would you have known Puka Nacua was going to be great? How would you have known Kellen Moore wasn't?"

The honest answer to both is: *you mostly couldn't*. Prospect projection has a hard floor of irreducible variance — NFL Draft hit rates have been the same ~50% for skill players for decades despite billions in scouting investment. Our goal is *not* to outsmart the league. Our goals are:

1. **Beat consensus on the players where the crowd is wrong for a *legible reason*** — small-school production, narrative-driven hype, low-SOS inflation.
2. **Surface "ghost prospects"** — players the crowd ignored but historical comps say they shouldn't have.
3. **Honest confidence bands** — when production gives us no signal (Puka Nacua had 1,749 career rec yds in 4 college years), the model should *say so* rather than fake a precise rank.

This is the same Kings-of-Dynasty consensus-vs-model framing as the NFL rankings: highlight where the data disagrees with KeepTradeCut, let the user weigh narrative themselves.

## 3. Methodology

### 3.1 Vectorization (modernized from v0.16)

The v0.16 engine z-scored players against the position-class corpus. **v3.0 uses fantasy-point arcs, matching v2.0+ NFL methodology** for consistency.

For each college player-season, compute:

```
v[0] = SOS-adjusted fantasy points per game (NFL ppr scoring applied to college stat line)
v[1] = team-share dominator (player rec yds + rush yds + pass yds) / team total
v[2] = team-share TD dominator
v[3] = age at end of season (younger = better signal)
v[4] = games played
v[5] = class (FR=1, SO=2, JR=3, SR=4, GR=5)
v[6] = years_to_breakout (first season with dominator >= 0.20)
v[7] = peak single-season fp/game
v[8] = career fp_per_game trajectory slope (improving = good)
v[9] = career total fp (volume anchor)
```

Then KNN within position (QB/RB/WR/TE) over the historical corpus.

### 3.2 SOS Adjustment (the Kellen Moore fix)

Each player-season's fantasy production gets multiplied by an SOS factor:

```
sos_multiplier = clip(
    1.0 + SOS_BETA * (team_season_sos - corpus_median_sos) / corpus_sd_sos,
    0.65,  # cap floor at FCS-tier baseline
    1.10,  # cap ceiling so SEC isn't infinitely boosted
)
```

- `SOS_BETA` calibrated empirically against the back-test: pick the value that maximizes correlation between adjusted-college-fp and realized-NFL-fp. Initial guess: `0.15`.
- This *reduces* Kellen Moore's effective production (Boise State's avg SOS 2008-2011 was ~3 points below FBS median) and *increases* a player who put up the same numbers against SEC.
- Also weights *opponent rank when they played* — bonus for production in games against top-25 opponents specifically. (Stretch goal — initial release uses season-average SOS.)

Combined with the existing `conference_tier` multiplier (`P5=1.0, G5_top=0.85, G5=0.75, FCS=0.65`), the SOS adjustment becomes a continuous refinement on top of the discrete conference bucket. Both terms stay because they capture different things: tier = "what league are you in", SOS = "who specifically did you play this year".

### 3.3 Bridge → NFL Outcome

For each college comp, look up their NFL career through `data/bridge/ncaa_to_nfl.json`. Comps with no NFL match are kept in the pool and contribute **zero realized fantasy points** — the bridge-miss rate is itself a strong negative signal. ("65% of your college comps didn't even make the NFL" is information.)

### 3.4 Prospect Score

```
prospect_score = weighted_average(
    realized_nfl_career_fp_through_year_3,
    weights=cosine_similarity_to_prospect,
    top_k=25,
)
```

Year-3 is the dynasty-rookie horizon — what matters for dynasty value is the early-career arc, not whether they eventually become a 12-year vet. Top-25 comps gives stability.

### 3.5 Confidence (the Puka Nacua honesty rule)

Confidence = inverse of cross-comp variance:
- **High confidence:** top-25 comps cluster tightly (CJ Stroud → fast-track elite QBs)
- **Low confidence:** comps are all over the map (Puka Nacua → "WRs with <2000 college rec yds who declared late after a transfer" is a tiny + noisy cluster)

Display as confidence band on the score, NOT a single number. Same `is_low_confidence` flag pattern v2.2 uses for the NFL engine.

This is the *honest* answer to the Nacua question: the model would *not* have ranked him highly, *but* its confidence in that ranking would have been very low. That's information — flag him as "unrankable on production alone" rather than confidently downgrading him.

### 3.6 What we explicitly DO NOT model (per Phil's spec)

- Athletic profile (40 time, vertical, broad, RAS score). The `data/ras/` corpus exists in the repo; we leave it on the table for v3.1 if back-test shows production alone misses too many Nacuas.
- Pro Day / Combine context.
- Scouting reports / draft media consensus.
- Coaching-tree effects.

The whole point is: *production-only baseline*. If a player ranks well purely on what they did on the field against a fair schedule, that's a signal independent of any narrative. Athletic context is its own separate model we can blend later.

## 4. Back-Test Plan

**Setup:** Run the model on every NFL skill-position player drafted 2002-2020 (gives them 5+ NFL seasons to materialize). Use *only* their college stats — pretend you're sitting in the draft room with zero NFL hindsight.

**Metrics:**
1. **Spearman rank correlation** between model's `prospect_score` and realized NFL career outcome (year-3 fantasy points). Compare to actual draft pick rank correlation as baseline. **Hypothesis: model beats draft order Spearman, because draft order is heavily contaminated by athletic profile + team need.**
2. **Hit rate at top-N:** of the prospects the model put in the top 10/25/50, what fraction became NFL fantasy-relevant (>= year-3 fp threshold by position).
3. **Calibration:** confidence-band coverage. Are 80% confidence intervals actually covering 80% of outcomes?
4. **Famous misses analysis:** the model's worst Type I errors (high score, NFL bust) and Type II errors (low score, NFL hit). Catalog them, look for systemic patterns. Puka Nacua and Kellen Moore are case studies #1 and #2.

**Output:** `docs/V3.0-BACKTEST.md` with the numbers + a "biggest misses" section as case studies.

## 6. Implementation Plan (numbered tasks)

**Six sequential PRs.**

### PR 1 — sports-reference scraper + 2000-2013 corpus extension
1. New module `src/dynasty/sources/sports_reference_cfb.py` — Wayback-Machine-backed scraper. Same shape as v2.4's PFR scraper. Polite throttle (1 req/s), HTML cache `data/sr_cache/{table}/{year}/{slug}.html`.
2. Scrape per-year position leaderboards (passing/rushing/receiving) for 2000-2013 to enumerate the player universe.
3. Scrape individual player pages for full career stats.
4. Normalize to the cfbfastR schema (`data/historical_ncaa_football/season_{year}.json` shape). Output: `season_2000.json` through `season_2013.json`, plus an `id_map.json` connecting cfbfastR `cfb_player_id` ↔ SR slug for the overlap years (2014+) where both sources exist.

### PR 2 — Strength of Schedule corpus
1. Scrape `years/{YYYY}-standings.html` for 2000-2025. Per-team `sos` + `srs`.
2. Compute `corpus_median_sos`, `corpus_sd_sos` for normalization.
3. Output: `data/sos/team_sos_{year}.csv` and a unified `data/sos/team_sos_all.csv.gz`.
4. Tests: SEC teams average sos > FCS teams; Alabama 2011 > Boise State 2011.

### PR 3 — College fantasy-arc engine
1. Resurrect & modernize `src/dynasty/similarity/` (deleted in d0cd216). Restore `vectorize.py`, `bridge.py`, `projection.py`, `rookie_projection.py`. **Rewrite vectorize to use the v2.x fantasy-point methodology** instead of z-scoring.
2. Apply SOS multiplier from PR 2.
3. Apply existing conference_tier multiplier from `historical_ncaa_football.py`.
4. Output: `data/engine_v3/college_fantasy_arc_corpus.json`. Same structure as `data/engine_v2/fantasy_arc_corpus.json`.

### PR 4 — Prospect projection + confidence
1. KNN engine: for any college player, return top-25 comps + similarity weights.
2. Resolve comps through `bridge/ncaa_to_nfl.json` → realized NFL career fp.
3. Compute `prospect_score`, `confidence_band` (low/med/high based on comp variance), `top_comp`, `bridge_miss_rate`.
4. Cache: `data/engine_v3/prospect_projections.json`.
5. Tests: known cases.
   - **CJ Stroud (2023)** should have high score + high confidence.
   - **Puka Nacua (2023)** should have low/medium score + **low confidence** (with reason: "limited college sample + transfer + below-median dominator").
   - **Kellen Moore (2012)** should have his Boise stats penalized by SOS, ending up medium score + high bust risk in his comp pool.
   - **Saquon Barkley (2018)** should be a top-3 RB prospect score.

### PR 5 — Back-test validation
1. Run the engine on every 2002-2020 NFL skill-position rookie using only their pre-draft college data.
2. Compute Spearman ρ vs draft order, hit-rate at top-N, calibration, biggest misses.
3. Doc: `docs/V3.0-BACKTEST.md`.
4. **Gate:** if model Spearman is *worse than* random within 1 SD, halt PR 6 and revisit methodology. The point of this gate is intellectual honesty — we ship a prospects tab only if the back-test validates it.

### PR 6 — Prospects tab UI
1. New page `dynasty_site/prospects.html` rendered from `src/dynasty/report.py`.
2. New KTC rookie consensus pull via `scripts/refresh_ktc_rookie_consensus.py`.
3. Crosswalk KTC rookies ↔ college player by (name, school).
4. Add "Prospects" nav link to the existing site header. Update methodology page.
5. **For the 2026 class specifically:** call out the top-25 with brief rationale. Highlight ghost prospects (low KTC, high model) and overhyped (high KTC, low model).

## 7. Risks

| Risk | Mitigation |
|---|---|
| Sports-reference Wayback coverage gaps | Fallback list: cfbfastR (we already have it) for 2014+, ESPN historical for older years, college football reference proper as last resort. |
| KTC rookie ranks contaminated by athletic profile | They are — that's the *point*. The Δ column will highlight exactly the players KTC is pricing on combine vs. production. |
| Position eligibility messiness | College "athlete" / "slot" / "h-back" labels are noisy. Normalize aggressively: anyone with >50% of touches as rushes → RB, etc. |
| Transfers (Puka WAS → BYU) | The cfbfastR data already handles this per-team-season. Aggregate at career level for vectorization, but keep per-team-season for SOS adjustment. |
| Players who barely played in college (Nacua at Washington) | Confidence band handles this. Don't pretend to rank them precisely. |
| Back-test contamination from draft-pick crosswalk leakage | The bridge uses (name, college, rookie_season ±1) — not draft pick. So the model can't cheat by looking at draft order. |
| **Big one: NFL draft itself is signal we're discarding** | True. We're explicitly building a production-only model so the Δ vs draft order is interpretable. v3.1 can blend in draft capital if back-test demands it. |

## 8. Scope Cuts

- **No combine / pro day / RAS in v3.0.** Phil explicit: production only. v3.1 question.
- **No play-by-play features** (yards-after-catch, target separation, etc.). Sports-reference doesn't expose them pre-2014. Stick to box-score stats.
- **No high school recruiting rankings.** Out of scope.
- **No "team success" features** (championships, bowls). Tempting because elite-team-context is real, but it conflates production with team strength which SOS already captures.
- **No 2025-season live data for 2026 prospects in the initial release.** Wayback caches are typically 6-12 months stale. We rank the 2026 class on their *through-2024* career, which is fine: it's exactly what any pre-season ranking would do. A live-season refresh script can come in v3.0.1 once we figure out the Cloudflare-bypass story (likely: browser tool with profile=user on a manual cadence).
